# SAP 同步失败处理规范

## 适用场景

当本地系统向 SAP 推送库存扣减、收货、退料、报工等数据失败时，适用本规范。

## 常见原因

1. SAP 接口超时。
2. SAP 返回业务校验失败。
3. 网络异常导致请求未完成。
4. 本地系统与 SAP 数据版本不一致。
5. 第三方集成服务重试次数达到上限。

## 处理原则

1. SAP 同步失败时，不应直接认为业务已成功。
2. 本地系统需要记录失败原因和重试次数。
3. 对于库存类操作，应优先保证账实一致。
4. 如果 SAP 未确认成功，本地系统应保留补偿或回冲能力。
5. 不建议在同步状态未知时继续执行后续库存扣减。

## 建议处置

1. 查询 SAP 同步日志。
2. 确认 SAP 是否已实际接收请求。
3. 如果 SAP 未成功，应执行重试或补偿。
4. 如果 SAP 已成功但本地未收到回执，应人工核对后修正状态。
5. 对库存类异常，应暂停后续自动扣减。
