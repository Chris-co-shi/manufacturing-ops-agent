# 库存扣减异常处理规范

## 适用场景

当工单投料、库存扣减、物料出库过程中出现扣减失败、库存不足、冻结库存异常、系统库存与现场库存不一致时，适用本规范。

## 常见原因

1. SAP 与本地系统库存同步失败。
2. 本地系统可用库存与现场实物库存不一致。
3. 库存已被冻结，但业务侧仍尝试扣减。
4. 并发扣减导致库存锁冲突。
5. AGV、WMS 或现场设备回传延迟，导致库存状态未及时更新。

## 处理原则

1. 不建议在库存状态不一致时继续扣减。
2. 应先暂停相关工单投料，避免异常扩大。
3. 应核对 SAP、本地系统和现场实物库存。
4. 如存在同步失败，应先确认失败原因，再执行补偿或重试。
5. 人工确认库存状态前，不建议自动恢复生产。

## 建议处置

1. 暂停当前工单继续投料。
2. 核对 SAP 与 XMOM 库存差异。
3. 检查冻结库存和可用库存。
4. 检查最近一次库存同步结果。
5. 确认现场实物库存。
6. 人工确认后再执行补偿或重新扣减。
